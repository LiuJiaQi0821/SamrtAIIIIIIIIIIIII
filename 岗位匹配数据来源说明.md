# 岗位匹配数据来源说明

## 📊 数据来源概述

岗位匹配的数据来源是 **Supabase PostgreSQL 数据库** 中的 `jobs` 表，包含约 **9958条真实招聘岗位数据**。

## 🔍 岗位搜索API详解

### 1. 搜索接口路径

```
POST /api/jobs/search
```

### 2. 支持的筛选条件

| 参数 | 说明 | 匹配方式 |
|------|------|---------|
| `industry` | 行业 | 模糊匹配 (ilike) |
| `keyword` | 关键词 | 同时匹配岗位名称/公司名称/职位描述 |
| `location` | 地点 | 模糊匹配 (ilike) |
| `salary` | 薪资 | 精确匹配 (eq) |
| `company` | 公司类型 | 精确匹配 (eq) |
| `size` | 公司规模 | 精确匹配 (eq) |
| `jobType` | 岗位类型 | 模糊匹配岗位名称 (ilike) |

### 3. 查询方式优化

**不使用 Supabase JS 客户端，直接调用 REST API**

原因：
- Supabase JS 客户端有默认限制（最多1000条）
- 需要获取所有符合条件的数据（可能超过1000条）
- REST API 更灵活，支持分页批量获取

**查询示例**：
```javascript
// 直接调用 Supabase REST API
const url = `${supabaseUrl}/rest/v1/jobs?select=*&industry=ilike.*互联网*`;

// 使用分页获取所有数据
// 第1页: limit=1000&offset=0
// 第2页: limit=1000&offset=1000
// ...直到获取完所有数据
```

## 🤖 AI匹配度打分

### 调用流程

```
从数据库获取 TOP 5 岗位
         │
         ▼
构建分析提示词：
- 学生画像数据
- 5个岗位的详细信息
- 评分规则和维度
         │
         ▼
调用 /api/chat (豆包大模型)
         │
         ▼
AI 进行匹配度分析：
- 基础要求匹配 (学历/经验)
- 职业技能匹配 (技术栈)
- 职业素养匹配 (软技能)
- 发展潜力匹配 (学习能力)
         │
         ▼
返回 JSON 格式的匹配分析结果
         │
         ▼
前端展示：
- 匹配度百分比
- 各维度分数
- 优势和差距
- 改进建议
```

### 评分维度

| 维度 | 权重 | 说明 |
|------|------|------|
| 基础要求匹配 | 自适应 | 学历、专业、工作年限 |
| 职业技能匹配 | 自适应 | 技术栈、工具、证书 |
| 职业素养匹配 | 自适应 | 沟通、抗压、团队协作 |
| 发展潜力匹配 | 自适应 | 学习能力、创新能力 |

**权重自适应规则**：
- 技术岗：职业技能权重增加
- 管理岗：职业素养和发展潜力权重增加
- 应届生：发展潜力权重增加

## 💡 核心要点总结

1. **数据来源**：Supabase PostgreSQL 数据库的 `jobs` 表
2. **数据量**：约 9958条真实招聘岗位数据
3. **查询方式**：直接调用 Supabase REST API（不使用 JS 客户端）
4. **分页策略**：每页1000条，批量获取所有符合条件的数据
5. **匹配策略**：6种逐步放宽的搜索策略，确保有结果
6. **AI打分**：豆包大模型进行四维度匹配度分析
7. **最终输出**：TOP 5 岗位，按匹配度从高到低排序
