# 项目上下文

## 技术栈

- **核心**: Vite 7, TypeScript, Express
- **UI**: Tailwind CSS

## 目录结构

```
├── scripts/            # 构建与启动脚本
│   ├── build.sh        # 构建脚本
│   ├── dev.sh          # 开发环境启动脚本
│   ├── prepare.sh      # 预处理脚本
│   └── start.sh        # 生产环境启动脚本
├── server/             # 服务端逻辑
│   ├── routes/         # API 路由
│   ├── server.ts       # Express 服务入口
│   └── vite.ts         # Vite 中间件集成
├── src/                # 前端源码
│   ├── index.css       # 全局样式
│   ├── index.ts        # 客户端入口
│   └── main.ts         # 主逻辑
├── index.html          # 入口 HTML
├── package.json        # 项目依赖管理
├── tsconfig.json       # TypeScript 配置
└── vite.config.ts      # Vite 配置
```

## 包管理规范

**仅允许使用 pnpm** 作为包管理器，**严禁使用 npm 或 yarn**。
**常用命令**：
- 安装依赖：`pnpm add <package>`
- 安装开发依赖：`pnpm add -D <package>`
- 安装所有依赖：`pnpm install`
- 移除依赖：`pnpm remove <package>`

## 开发规范

- 使用 Tailwind CSS 进行样式开发

### 编码规范

- 默认按 TypeScript `strict` 心智写代码；优先复用当前作用域已声明的变量、函数、类型和导入，禁止引用未声明标识符或拼错变量名。
- 禁止隐式 `any` 和 `as any`；函数参数、返回值、解构项、事件对象、Express `req`/`res`、`catch` 错误在使用前应有明确类型或先完成类型收窄，并清理未使用的变量和导入。

## 问题或错误及解决方案

### 问题: 岗位搜索API只返回50条数据
- **原因**: Supabase JS 客户端有默认限制（1000条），但原代码只获取第一页
- **解决方案**: 改用 REST API 直接调用，使用分页查询（每页1000条）获取所有符合条件的数据（数据库共9958条）

### 问题: 薪资筛选不生效
- **原因**: 
  1. 后端代码未处理 salary 参数
  2. 前端薪资选项格式（如 "10K-20K"）与数据库格式（如 "1-2万"）不匹配
- **解决方案**: 
  1. 在后端添加 salary 筛选逻辑
  2. 修改前端薪资选项，使用与数据库匹配的格式

### 薪资筛选前端选项（与数据库匹配）
```typescript
salary: ['3000-4000元', '4000-6000元', '5000-7000元', '6000-8000元', '7000-10000元', '8000-12000元', '1-1.5万', '1.5-2万', '2-3万', '3万以上']
```

### 新增筛选条件
1. **公司规模** (size): 20人以下, 20-99人, 100-299人, 300-499人, 500-999人, 1000-9999人, 10000人以上
2. **岗位类型** (jobType): C/C++, Java, Python, 前端开发, 后端开发, 算法工程师, 产品经理, 数据分析师, 测试工程师, 运维工程师, UI设计, 运营

### 关键词搜索功能
- 用户可以输入关键词搜索岗位
- 关键词同时匹配岗位名称、公司名称、职位描述
- 支持与筛选条件组合使用
- 后端使用 PostgREST `or` 条件实现多字段模糊匹配

### 薪资显示优化
- 薪资字段中的 "·XX薪" 等额外内容在前端显示时自动去除
- 使用 `formatSalary()` 函数处理，显示如 "1.2-1.8万" 而不是 "1.2-1.8万·15薪"

### UI样式优化
- 岗位搜索结果卡片与文件发送按钮保持一致的视觉风格
- 使用柔和的浅蓝色渐变背景
- 卡片带有蓝色光影效果的阴影
- 标签使用统一的圆角胶囊样式
- 筛选标签与文件上传按钮保持一致的浅蓝色渐变背景和光影效果
- 搜索按钮和关键词输入框也采用统一的浅蓝色渐变风格
